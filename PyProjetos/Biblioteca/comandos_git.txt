*******************************************************************
🧠 Fluxo básico (decora isso)

Esse é o caminho padrão no dia a dia:

git status
git add .
git commit -m "mensagem clara"
git push

*******************************************************************

Comandos básicos (uso diário)

git init
> Inicializa um repositório Git na pasta atual.
Use uma vez, quando o projeto ainda não tem Git.

git status
> Mostra o estado atual do repositório:
arquivos modificados
arquivos não rastreados
o que está pronto para commit
👉 É o comando mais usado.

git add .
> Adiciona todos os arquivos modificados à área de preparação (stage).

Diz ao Git: “esses arquivos entram no próximo commit”.

📌 Também existe:
git add nome_do_arquivo.py

git commit -m "mensagem"
> Cria um registro oficial das mudanças.
A mensagem deve explicar o que foi feito.

Exemplo:
git commit -m "Adiciona exercícios de lógica em Python"

git log
> Mostra o histórico de commits:
autor
data
mensagem
hash

git log --pretty=oneline (mais usado e relação ao git log)
> Exibe o histórico de commits de forma resumida, mostrando um commit por linha, com hash + mensagem.

👉 Útil pra acompanhar a evolução do projeto.

🔹 Comandos de conexão com GitHub

git branch
> Mostra as branches existentes.
A branch principal costuma ser main.

git branch -M main
> Renomeia a branch atual para main.

Muito usado em projetos novos.

git remote add origin URL
> Conecta o projeto local a um repositório no GitHub.

Exemplo:
git remote add origin https://github.com/seu-usuario/seu-repo.git

git push -u origin main
> Envia os commits locais para o GitHub.

-u define o padrão
depois disso, basta usar git push

git pull
> Atualiza o projeto local com as mudanças do GitHub.

Use quando trabalha em mais de um computador.

git clone URL
> Copia um repositório do GitHub para o seu PC.

Exemplo:
git clone https://github.com/seu-usuario/seu-repo.git

🔹 Comandos de comparação e correção

git diff
> Mostra o que mudou no código antes do commit.

git checkout -- arquivo
> Descarta mudanças em um arquivo específico.

⚠️ Cuidado: não tem volta.

🚀 Como tirar um arquivo da área do commit?

git restore
> Remove o arquivo da staging área, mas mantém suas alterções no arquivo.

Exemplo:
git restore --staged arquivo.py

git reset
> Remove arquivos do stage sem apagar o código.

Exemplo:
git reset arquivo.py

🔹 Comandos úteis para organização

git rm arquivo
> Remove um arquivo do projeto e do controle do Git.

git mv antigo.py novo.py
> Renomeia arquivos mantendo o histórico.

✅ Como corrigir io terminal está mostrando os acentos em codificação UTF-8. (mostrar acentos normalmente)

Execute este comando no terminal:

git config --global core.quotepath false

O que esse comando faz:
core.quotepath false 
> faz o Git mostrar os nomes com acentos normalmente.

--global
> aplica para todos os seus repositórios.

✅ Como resolver de vez
👉 Passo 1 — adicionar seus arquivos .c
git add .

ou (melhor prática):

git add *.c

🧠 Dica de ouro (nível profissional)

Se quiser ver tudo sempre:

git status -uall

🚨 CALMA — COMO VOLTAR AO NORMAL

Você quer voltar exatamente como estava antes.

✅ Solução rápida (perfeita pra isso):
git restore --staged .

O que esse comando faz:

👉 Ele desfaz tudo que está em verde
👉 Volta os arquivos para o estado anterior

💡 Forma correta pro seu caso (mais segura)

Pra ignorar só .exe, o ideal era:

git rm --cached *.exe

👉 Bem mais seguro — não mexe no resto

🔥 Resumo (guarda isso)
Comando	O que faz
git rm --cached *.exe	remove só exe do Git
git rm -r --cached .	remove TUDO do Git ⚠️
git restore --staged .	desfaz bagunça no stage ✅
